# 安全策略

## 报告漏洞

请不要在公开 Issue 中披露未修复漏洞。使用仓库 **Security → Report a vulnerability** 私密报告功能，并提供受影响版本、复现步骤、可能影响和建议修复方式。

若仓库尚未启用私密漏洞报告，请联系仓库所有者建立安全沟通渠道，不要公开附带漏洞细节。

## 敏感信息

不得提交：

- `.env` 与生产数据库连接串；
- JWT、GitHub、域名或云服务密钥；
- PostgreSQL 备份；
- 玩家访问令牌、恢复码或完整生产日志。

生产 `JWT_SECRET` 至少使用 32 字节随机值，并在疑似泄露时立即轮换。恢复码只保存摘要，明文仅展示一次。

## 支持范围

安全修复优先应用于 `main`。项目处于首版阶段，不承诺为历史提交提供长期补丁。

## 已知边界

排行榜采用服务端路径重放与实用级时间校验，但不能阻止共享答案、屏幕外求解或专业自动化客户端。不要将当前系统用于奖金赛事或强身份竞赛。
